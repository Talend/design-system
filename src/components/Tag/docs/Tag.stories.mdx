## When to use this component

Use the tag when you need to display a status or set of discrete attributes.

An example could be when an item in a task list has been ‘completed’, or an object has multiple labels to help with search and discoverability.

## When not to use this component

Don't use this component inside body content or as a way to highlight sections of content.

## Using this component

Tags work best when they are short adjectives. Verbs should be avoided as they may cause the user to think the component is a link. Tags should always be written in sentence case.

## Variations

### Contextual tags

The tag component supports a set of generic contexts including:
- `Information`
- `Success`
- `Warning`
- `Destructive`

<Preview>
  <Story name="Information">

	</Story>
  <Story name="Success">
    <Tag.Success>Success</Tag.Success>
	</Story>
  <Story name="Warning">
    <Tag.Warning>Warning</Tag.Warning>
	</Story>
  <Story name="Destructive">
    <Tag.Destructive>Destructive</Tag.Destructive>
	</Story>
</Preview>

### Truncated tags

Tag length is always truncated to a proportional length that is relative to the font size used. Truncated tags will be appended with an ellipsis (...).

    <Tag>Awaiting test completion</Tag>
	</Story>
 </Preview>

## Accessibility

Large collections (clouds) of tags may be a source of visual confusion for some users. If you expect to display more than 10-15 tags in a single context, it may be appropriate to give users the option of displaying tags as a list.
